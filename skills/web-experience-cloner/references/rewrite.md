# Phase 7 · 从零重写特效（算法级 1:1）

不依赖原框架/原代码，用实捕 GLSL 作"真值"独立重建。这是**真正证明理解**的一步。

## 选型：挑最自包含的
| 易 → 难 | 类型 | 例 |
|---------|------|----|
| ★ | 2D 全屏单 pass shader | LUT/色彩分级、噪声场、暗角 |
| ★★ | 2D ping-pong 模拟 | curl 流体画笔、反应扩散、热扩散 |
| ★★ | Canvas2D / DOM+GSAP | 翻牌、圆度度量、文字揭示 |
| ★★★ | 3D PBR / 后处理链 | 需重建 IBL/Gobo/TAA，建议保留 Three.js |
| ★★★★ | 高斯泼溅 / 光场 | 用官方 loader，别手撸格式解码 |

## 模板：原生 WebGL2 + ping-pong FBO
1. 全屏三角形 + 两个 `RGBA16F` FBO 互换。
2. **update pass**：逐行搬运实捕的 frag（保持 `sdSegment`/`noised`/平流/耗散等语义）。
3. **display pass**：把模拟纹理可视化（这步可自由设计，原站常把它当画面位移用）。
4. 指针/时间用与原站一致的驱动（如内联 `SecondOrderDynamics`）。
参考实现见本技能配套示例（curl-paint）。

## 关键坑（真实踩过）
- **参数依赖纹理尺度**：平流位移 = `velInv * texelSize`，所以 `pushStrength` 往往要比直觉大 10×+，
  否则"注入了但不流动"。一开始用小值 → 墨水只堆不动，放大后才正常。
- **耗散系数**贴近 1（如 0.99x）才有持久拖尾；太小则瞬间消失。
- **色彩空间**：全程线性计算，仅最终输出转 sRGB（对照原站 `aces`/`colorspace` chunk）。
- **时间基**：用与原站一致的 dt 来源，别用墙钟混合。
- **验证靠截图**：用 `evaluate_script` 程序化划线/推进，再 `take_screenshot` 看行为对不对，
  而不是凭空调参。停驻应累积变亮、快扫应留卷曲拖尾——行为对了才算复刻成功。

## 诚实边界
display 风格与原站可不同（算法一致即可），文档里写清"算法 1:1、呈现风格为 demo 重设计"。
