# Phase 4 · 文档模板

## 每个组件一篇，结构固定
```markdown
# NN · 组件名（页面上的标题）

截图：recon/shots/xx.png。类：ClassA / ClassB（deobf 行号）。
DOM：#id-…。着色器：progNN（recon/shaders/progNN.frag.glsl）。uniform：u_xxx…

## 视觉
（看截图客观描述：看到了什么、怎么动、怎么交互）

## 原理
（分点拆解。证据分两档标注：
  - 实捕：着色器全文/uniform 名/类实现 → 写“实捕”
  - 推断：根据命名/常识推测 → 写“推断”）
（涉及算法给伪代码或逐行映射到捕获的 GLSL）

## 复刻要点
| 部分 | 难度 | 做法 |
```

## 另写两篇横切文档
- **架构总览**：技术栈表、顶层对象图、启动初始化链、公共 GLSL chunk 库、后处理链顺序、
  滚动系统、Section 基类、**组件→文档索引表**。放最前面（00）。
- **通用动画引擎**：把被各组件复用的数学/动画原语集中讲（二阶动力学、Wiggle、布朗、缓动、
  滚动区间映射）。各组件文档引用它，避免重复。

## 风格约定
- 语言跟用户（中文/英文），全系列统一。
- 行号、uniform 名、文件路径都写出来——可点击、可复查。
- 诚实：实捕 vs 推断分清；做不到的（如某资产没下到）直说。
- “梗/叙事”与“技术”分开写（创意站常把技术包装成产品卖点，别混淆）。
